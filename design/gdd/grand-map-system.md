# Grand Map System (大地图系统)

> **Status**: Draft Complete
> **Author**: user + agents
> **Last Updated**: 2026-06-01
> **Implements Pillar**: Pillar 4 (Readable Pressure — 地图让威胁可视化)

## Overview

大地图系统管理游戏的空间层——由州（省级）和郡（战斗级）组成的双层区域地图。逻辑层是命名区域的邻接图，表现层是风格化三国历史地图。战斗、部署、资源产出都发生在郡级。控制一个州的全部郡解锁州级特殊能力（如特定兵种招募权），为玩家创造"集齐全州"的战略动力。无建筑系统——区域只有地形、产出和控制权。

## Player Fantasy

"我已经拿下荆州三个郡中的两个了——只差江夏！一旦拿下，荆州的水军招募权就是我的。但北边曹操正在集结兵力南下……我是先完成荆州还是回防？"

州级能力让地图不只是"占地盘"，而是"集齐一套解锁奖励"——卡牌收集的快感延伸到了地图上。

## Detailed Design

### Core Rules

#### 1. Map Structure (地图结构)

双层结构：

```
Map
├── 州 (Province) × 8-9
│   ├── 郡 (Commandery) × 3-4 per province
│   └── province_ability: 控制全州解锁的特殊能力
└── 邻接关系 (Adjacency Graph): 郡与郡之间的连接
```

| **徐州** | 3 | 东南要冲，商路交汇 | 解锁：计策卡获取费-30% |

| **荆州** | 3 | 南北咽喉，兵家必争 | 解锁：水军招募权 + 地形标签"水战地利"对全州生效 |

#### 3. Commandery Definition (郡级区域定义)

每个郡的数据结构：

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识（如"jingzhou_xiangyang"） |
| `name` | string | 显示名称（如"襄阳"） |
| `province` | string | 所属州 |
| `terrain` | enum | 地形类型（平原/山地/江河/城池/要塞） |
| `income_value` | int | 每回合粮草产出 |
| `controller` | enum | 当前控制者（player/ai_faction_id/neutral） |
| `adjacent` | array[string] | 邻接郡列表 |
| `garrison` | StackRef | nullable | 驻守的Stack（如有） |

#### 4. Commandery Table (郡级区域示例 — 荆州)

| 郡 | 地形 | 产出 | 邻接 | 历史参考 |
|----|------|------|------|----------|
| 襄阳 | 城池 | 20 | 南阳, 江夏, 长沙 | 荆州治所，刘表/刘备据点 |
| 江夏 | 江河 | 15 | 襄阳, 长沙, 柴桑(扬州) | 孙权与刘备争夺焦点 |
| 长沙 | 平原 | 15 | 襄阳, 江夏, 零陵(益州方向) | 荆南四郡之一 |

*（完整30郡表在实现时由内容设计填充，此处仅展示结构）*

#### 5. Adjacency Rules (邻接规则)

- 只有相邻的郡之间可以移动/进攻
- 跨州的郡也可以相邻（如荆州·江夏 ↔ 扬州·柴桑）
- 邻接关系是双向的
- 无"飞地"——所有区域通过邻接连通

#### 6. Province Control (州级控制)

```
province_controlled(province) → bool:
  return all(commandery.controller == player for commandery in province.commanderies)
```

控制全州后：
- 立即解锁该州的特殊能力
- UI上显示州级标记（金色边框/旗帜）
- 失去任何一个郡 → 立即失去州级能力

#### 7. Region Control Change (区域控制权变更)

- 攻击方在自动战斗中获胜 → 该区域控制权转移给胜方
- 进攻空区域（无驻军） → 直接占领
- 失去区域 → 下回合INCOME减少该区域产出

#### 8. Fog of War (战争迷雾 — MVP简化)

- MVP: 全地图可见，但敌方Stack内部组成不可见（只能看到"这里有一支军队"）
- 侦查计策卡可揭示敌方Stack详情
- 符合Pillar 4: 威胁位置可见，但具体强度需侦查

### States and Transitions

区域状态：

| 状态 | 说明 |
|------|------|
| `neutral` | 无人控制，可直接占领 |
| `player_controlled` | 玩家控制，产出粮草 |
| `ai_controlled` | AI控制，需战斗夺取 |
| `contested` | 本回合有进攻部署指向此区域（视觉提示） |

```
neutral → [玩家占领] → player_controlled
neutral → [AI占领] → ai_controlled
player_controlled → [AI攻占] → ai_controlled
ai_controlled → [玩家攻占] → player_controlled
any → [进攻部署指向] → contested (临时视觉状态)
```

### Interactions with Other Systems

| 系统 | 交互方式 |
|------|----------|
| **地形修正系统** | 提供区域terrain_type |
| **资源经济系统** | 提供区域income_value + 州级粮草加成 |
| **回合部署系统** | 提供可部署的目标区域列表（邻接限制） |
| **AI对手系统** | 提供AI控制区域、邻接信息、战略价值 |
| **自动战斗系统** | 战后更新区域控制权 |

## Formulas

### 玩家总收入

```
player_total_income = BASE_INCOME 
  + Σ(controlled_commanderies[i].income_value)
  + Σ(province_bonuses)  # 州级能力中的产出加成

示例（控制荆州全境 + 豫州2/3）:
= 20 + (20+15+15) + (15+12+0) + 0  = 20 + 50 + 27 = 97
# 荆州全控但荆州能力是水军权不是粮草加成
```

### 州控制判定

```
is_province_complete(province) → bool:
  return count(c for c in province.commanderies if c.controller == player) 
         == len(province.commanderies)
```

### 区域战略价值（供AI使用）

```
strategic_value(commandery) = 
  commandery.income_value 
  + (20 if would_complete_province else 0)  # 集齐全州的额外价值
  + (10 × count(adjacent_player_controlled))  # 威胁玩家的位置价值
```

## Edge Cases

| # | 边界情况 | 处理规则 |
|---|----------|----------|
| 1 | 玩家失去一个郡导致州级能力消失 | 立即生效。如果该能力是"水军招募权"，已有的水军不消失，但不能再招新的 |
| 2 | 玩家控制的区域被完全包围（飞地） | 仍然保持控制权和产出，但无法从该区域发起进攻（无邻接己方区域可出发）。敌方可以进攻它 |
| 3 | 同一回合多个AI同时进攻同一个玩家区域 | 按AI优先级排序，第一个AI先战斗。如果第一个AI赢了，第二个AI与该AI战斗（非玩家参与） |
| 4 | 玩家0区域（全部丢失） | 游戏失败条件触发 |
| 5 | 地图初始状态——哪些区域属于谁 | 由剧本系统定义。每个剧本设定初始控制权分布 |
| 6 | 两个不相邻的区域之间能否移动 | 不能。必须通过邻接链逐步推进（每回合移动1郡） |

## Dependencies

**上游依赖**：

| 系统 | 依赖内容 |
|------|----------|

| 系统 | 依赖性质 |

## Tuning Knobs

| 参数名 | 当前值 | 安全范围 | 影响 |
|--------|--------|----------|------|
| `TOTAL_COMMANDERIES` | ~30 | 20-40 | 区域总数。太少→缺乏战略纵深；太多→管理负担 |
| `COMMANDERIES_PER_PROVINCE` | 3-4 | 2-5 | 每州郡数。太少→州级能力太容易获得；太多→集齐困难 |
| `PROVINCE_ABILITY_STRENGTH` | 中等 | 弱-强 | 州级能力强度。太强→必须先集齐州才能竞争；太弱→无动力集齐 |
| `INCOME_RANGE` | 8-30 | 5-40 | 区域产出范围。差距大→富饶区域成争夺焦点 |
| `PLAYER_STARTING_REGIONS` | 3 | 2-5 | 玩家初始区域数。决定开局自由度 |

## Visual/Audio Requirements

- 地图以风格化三国历史地图呈现（水墨/手绘风格）
- 州级边界用淡色虚线标记
- 玩家控制区域用主色调（朱红/金）填充
- AI控制区域用对应势力色填充
- 中立区域灰色
- 州级能力解锁时：金色边框动画 + 成就音效
- 区域切换控制权时：颜色渐变动画

## UI Requirements

- 地图支持缩放：全局视图（看州级分布）↔ 局部视图（看郡级详情）
- 点击区域显示详情面板：名称、地形、产出、驻军、邻接区域
- 州级面板：显示当前控制进度（如"荆州 2/3"）和解锁能力预览
- 敌方区域显示威胁指示器（Pillar 4: 可预见压力）
- 邻接高亮：选中一个区域时，高亮所有可达的邻接区域
