# Terrain Modifier System (地形修正系统)

> **Status**: Draft Complete
> **Author**: user + agents
> **Last Updated**: 2026-05-31
> **Implements Pillar**: Pillar 2 (Decision Before Action — 部署前就能看到地形影响)

## Overview

地形修正系统定义地图区域的地形类型如何影响在该区域发生的战斗。影响分两层：（1）地形为参战Stack添加临时标签，可能触发武将技能联动（主要机制）；（2）地形对特定兵种提供小幅百分比战力修正（保底感知）。地形是区域的固有属性，不是卡牌——玩家不持有、不拖拽地形，而是通过选择"在哪里打"来利用地形。

## Player Fantasy

"我把周瑜部署到江河区域防守——因为江河地形会给他'水战地利'标签，完美触发他的火计技能。而敌方骑兵冲过来还会被水域削弱。地形就是我的第二层武器。"

## Detailed Design

### Core Rules

#### 1. Terrain Types (地形类型)

MVP共5种地形：

| 地形 | 说明 | 授予标签 | 标签接收方 |
|------|------|----------|------------|
| **平原** | 开阔地带，利于骑兵突击 | `平原地利` | 攻击方 |
| **山地** | 险要地势，利于防守设伏 | `山地地利` | 防守方 |
| **江河** | 水域地带，利于水军作战 | `水战地利` | 防守方 |
| **城池** | 城墙防御，利于步兵坚守 | `城池坚守` | 防守方 |
| **要塞** | 军事要塞，攻城器械有优势 | `要塞攻坚` | 攻击方 |

**规则**：
- 每个地图区域有且仅有1种地形类型
- 地形标签是临时的——仅在该区域的战斗期间生效，战斗结束后自动移除
- 攻击方/防守方由回合部署系统决定（主动进攻=攻击方，驻守本区域=防守方）

#### 2. Terrain Tags Integration (标签集成)

地形标签在战斗开始时注入到对应Stack的有效标签集中：

```
combat_tags(stack, terrain, role) = stack_tags(stack) ∪ terrain_tags(terrain, role)
```

这些临时标签参与技能联动系统的触发判定。

**设计示例**：
- 周瑜技能条件：`tag_present("水战地利")` → 在江河防守时自动触发
- 吕布技能条件：`tag_present("平原地利")` → 在平原进攻时自动触发
- 这意味着同一个武将在不同地形会有不同表现——部署选择更有深度

#### 3. Base Modifiers (基础修正)

除标签外，地形对兵种提供小幅战力修正（确保所有Stack都能感知地形）：

| 地形\兵种 | 步兵 | 骑兵 | 弓兵 | 水军 | 器械 |
|-----------|------|------|------|------|------|
| **平原** | 0% | +10% | 0% | -10% | 0% |
| **山地** | +5% | -10% | +10% | -15% | -10% |
| **江河** | -5% | -15% | -5% | +15% | -20% |
| **城池** | +10% | -5% | +5% | 0% | +15% |
| **要塞** | +5% | -10% | 0% | 0% | +10% |

**修正应用目标**：该兵种贡献的power值（非整个Stack）。
**修正不叠加**：每个兵种只受当前战斗区域的地形影响。

#### 4. Terrain Data Location (数据归属)

地形数据属于大地图系统中的区域定义：

```
Region {
  id: "jingzhou"
  name: "荆州"
  terrain: "river"  ← 本系统读取此字段
  ...
}
```

本系统不拥有数据，只定义"地形→效果"的映射规则。

---

### States and Transitions

地形修正系统无内部状态——它是纯查询函数：

```
输入: terrain_type + attacker_stack + defender_stack
输出: {
  attacker_bonus_tags: [...],
  defender_bonus_tags: [...],
  troop_modifiers: { troop_instance_id: modifier_percent, ... }
}
```

每次战斗重新计算，不存储历史。

---

### Interactions with Other Systems

| 系统 | 交互方式 |
|------|----------|
| **大地图系统** | 提供区域的terrain_type数据 |
| **卡牌数据系统** | 读取兵种的troop_type以确定基础修正 |
| **技能联动系统** | 地形临时标签注入后，技能联动正常判定（无需额外接口） |
| **自动战斗系统** | 调用本系统获取修正数据，应用到战斗计算中 |

## Formulas

### 地形临时标签注入

```
terrain_tags(terrain, role) → set[string]:
  match (terrain, role):
    ("plains", "attacker")  → {"平原地利"}
    ("mountain", "defender") → {"山地地利"}
    ("river", "defender")   → {"水战地利"}
    ("city", "defender")    → {"城池坚守"}
    ("fortress", "attacker") → {"要塞攻坚"}
    _                        → {}  (其他组合不授予标签)
```

### 兵种基础修正

```
terrain_modifier(terrain, troop_type) → float:
  return TERRAIN_MODIFIER_TABLE[terrain][troop_type]  // 查表

## 应用到战力计算
modified_troop_power(troop, terrain) = troop.power × (1 + terrain_modifier(terrain, troop.troop_type) / 100)
```

### 示例

```
场景: 周瑜Stack进攻荆州(江河地形)

周瑜(power=60) + 水军×2(power=10each) + 弓兵×1(power=12)

地形标签: 攻击方在江河 → 无标签授予（江河标签给防守方）
基础修正: 水军+15%, 弓兵-5%

水军1修正后: 10 × 1.15 = 11.5
水军2修正后: 10 × 1.15 = 11.5
弓兵修正后: 12 × 0.95 = 11.4

Stack总战力 = 60 + 11.5 + 11.5 + 11.4 = 94.4

注意: 周瑜的"水战地利"技能条件不满足（他是攻击方，标签给了防守方）
→ 如果周瑜改为在江河防守，他会获得"水战地利"标签，技能触发！
```

## Edge Cases

| # | 边界情况 | 处理规则 |
|---|----------|----------|
| 1 | 双方都是进攻者（两个Stack同时进攻同一区域） | 先到者视为"防守方"（占据地利），后到者为攻击方。若同回合同时进攻空区域→无人获得防守标签 |
| 2 | Stack中混合多种兵种（骑兵+步兵+弓兵） | 每种兵种分别计算各自的修正，互不影响 |
| 3 | 地形标签与卡牌自带标签重名 | 不可能发生——地形标签有专用命名（"平原地利"/"水战地利"等），不与兵种/武将标签重名 |
| 4 | 区域地形变化（如计策卡改变地形） | MVP不支持地形变化。区域地形在剧本中固定。Future: 可通过special计策临时覆盖 |
| 5 | 修正后power出现小数 | 取整：四舍五入到整数 |
| 6 | 修正使power降到0以下 | 不可能：最大负修正-20%，最小power=5 → 最低=4 |

**下游依赖方**：

| 系统 | 依赖性质 |

## Tuning Knobs

| 参数名 | 当前值 | 安全范围 | 影响 |
|--------|--------|----------|------|
| `BASE_MODIFIER_MAX` | ±15% | ±5-25% | 基础修正的最大值。过大→地形决定论；过小→玩家无感 |
| `TERRAIN_TAG_GRANT_RULE` | 单方授予 | 单方/双方 | 当前只给一方标签。若改为双方→战术深度增加但复杂度上升 |
| `TERRAIN_TYPE_COUNT` | 5 | 3-8 | 地形种类数。太少→地图同质；太多→记忆负担 |

**调参建议**：先测试纯标签触发（无基础修正）→ 如果玩家反馈"地形没感觉"再逐步调高基础修正。

## Visual/Audio Requirements

- 地图上每个区域需清晰展示地形类型（图标+底色）
- 部署预览时，目标区域的地形效果需高亮显示（如："江河 → 防守方获得水战地利"）
- 战报中地形修正需标注（如："[地形] 水军+15%"）

## UI Requirements

- 区域详情面板显示：地形类型 + 对各兵种的基础修正表 + 授予的标签
- 部署阶段选择目标区域时，显示"地形提示"tooltip
- 如果Stack中的武将技能条件包含地形标签，在部署预览中高亮提示"该地形可触发技能"

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|----------|----------|
| 1 | 5种地形各自正确授予对应标签给正确的一方 | 单元测试 |
| 2 | 地形临时标签在战斗后不残留在Stack上 | 单元测试 |
| 3 | 兵种基础修正正确查表并应用 | 单元测试（每种地形×每种兵种） |
| 4 | 混合兵种Stack中每种兵种分别计算修正 | 单元测试 |
| 5 | 小数结果正确四舍五入 | 单元测试 |
| 6 | 地形标签正确参与技能触发判定 | 集成测试（周瑜在江河防守→技能触发） |
| 7 | 部署预览正确显示地形效果提示 | 手动测试 |
